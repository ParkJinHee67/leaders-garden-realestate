## 1. 시스템 개요

이 홈페이지는 **화이트라벨(White-label) 방식**으로 설계된 부동산 중개소 전용 플랫폼입니다.  
별도의 코딩 없이 **관리자 페이지에서 모든 정보를 입력하면** 홈페이지에 즉시 반영됩니다.

### 주요 기능 목록

| 기능 | 설명 |
|------|------|
| 🖼️ 대문(메인) 이미지 | 사무실/배경 사진 업로드 |
| 💬 슬로건 문구 | 메인/서브 슬로건 자유 편집 |
| 🕐 영업시간 | 평일/토/일/공휴일 개별 설정 |
| 📢 특별 공지 | 휴가, 명절 등 실시간 공지 |
| 📱 명함 이미지 | 대표자 명함/사진 업로드 |
| 🏠 매물 관리 | 매물 등록/수정/삭제 |
| 📋 상담 신청 | 고객 상담 신청 내용 확인 |
| 🗺️ 지도 연동 | 네이버 지도 사무소 위치 표시 |
| 📧 이메일 알림 | 상담 신청 시 자동 이메일 발송 |

## 2. 관리자 로그인

### 접속 방법

1. 홈페이지 주소 뒤에 `/admin` 을 입력합니다.  
   예: `https://내홈페이지주소.com/admin`

2. 관리자 **비밀번호**를 입력하고 로그인합니다.

> ⚠️ **주의**: 비밀번호는 외부에 절대 공유하지 마세요.  
> 비밀번호 변경이 필요한 경우 시스템 관리자에게 문의하세요.

## 3. 사이트 환경 설정 (핵심)

관리자 로그인 후 **[사이트 환경 설정]** 탭을 클릭합니다.

### 3-1. 기본 정보

| 항목 | 입력 예시 | 설명 및 표시 위치 |
|------|-----------|------------------|
| 상호명 | 리더스가든 부동산 | 헤더, 푸터 상호명 표시 |
| 대표자명 | 유현휘 | 푸터 대표자명 표시 |
| 대표 연락처 (고객 노출용) | 010-4630-0363 | 헤더 및 푸터 고객센터 번호 (고객 노출용) |
| 알림 수신 전화번호 (SMS 알림용) | 010-1234-5678 | 상담 신청이 들어올 때 알림 문자를 받을 핸드폰 번호 (사설 관리용) |
| 알림 수신 이메일 | email@gmail.com | 상담 신청이 들어올 때 알림 메일을 수신할 이메일 주소 |
| 등록번호 | 123-45-67890 | 중개업소 개설등록번호 (푸터 표시용) |
| 사무실 주소 | 경기도 양주시 옥정로... | 푸터 및 네이버 오시는길 지도 동적 매칭에 사용 |

### 3-2. 사이트 이미지 설정

#### 대표(명함) 이미지
- 홈페이지 중간 **"대표 소개"** 영역에 크게 표시됩니다.
- 명함 사진, 프로필 사진 모두 사용 가능합니다.
- 권장 형식: JPG, PNG | 권장 크기: 600×400px 이상

#### 메인 배경(히어로) 이미지
- 홈페이지 최상단 **대형 배경 이미지**로 표시됩니다.
- 사무실 내부, 단지 전경 사진 등을 활용하세요.
- 권장 형식: JPG | 권장 크기: 1920×600px 이상

**업로드 방법**:
1. [파일 선택] 버튼 클릭 → 사진 선택
2. 하단 **[환경 설정 저장하기]** 버튼 클릭
3. "이미지 업로드 및 저장 중..." 메시지가 사라지면 완료

### 3-3. 문구 및 슬로건

#### 메인 슬로건 (큰 글씨)
- 배경 이미지 위에 크게 표시되는 핵심 문구입니다.
- **[-- 예시에서 선택 --]** 드롭다운에서 미리 만들어진 예시를 골라 쓸 수 있습니다.
- 줄바꿈은 Enter로 입력하면 홈페이지에도 그대로 반영됩니다.

#### 서브 슬로건 (작은 글씨)
- 메인 슬로건 아래에 작게 표시되는 부가 설명 문구입니다.
- 드롭다운에서 예시 선택 후 원하는 대로 수정 가능합니다.

### 3-4. 영업시간 및 공지사항

| 항목 | 입력 예시 |
|------|-----------|
| 평일 | 09:00 ~ 19:30 |
| 토요일 | 09:00 ~ 17:00 |
| 일요일 | 휴무 (사전 예약 시 상담 가능) |
| 공휴일 | 휴무 (사전 예약 시 상담/탐방 가능) |
| 특별 공지사항 | 5/25일 정상근무합니다. |

- 영업시간은 홈페이지 **하단(푸터) 고객센터** 영역에 자동으로 표시됩니다.
- **특별 공지사항**을 입력하면 푸터에 **주황색 강조 박스**로 눈에 잘 띄게 표시됩니다.
- 명절이나 휴가가 끝난 후에는 공지사항 내용을 **비워두면** 자동으로 사라집니다.

### 3-5. API 환경 변수 세팅

> ⚠️ 이 항목은 **최초 1회**만 설정하면 됩니다. 함부로 변경하지 마세요.

| 항목 | 설명 |
|------|------|
| 네이버 지도 Client ID | 지도에 사무소 위치를 표시하기 위한 키 |
| EmailJS Service ID | 이메일 발송 서비스 ID |
| EmailJS Template ID | 이메일 양식 ID |
| EmailJS Public Key | 이메일 서비스 인증 키 |

## 4. 매물 관리

관리자 로그인 후 **[매물 관리]** 탭을 클릭합니다.

### 4-1. 매물 등록

1. 상단 **[새 매물 등록]** 섹션에서 정보를 입력합니다.

| 입력 항목 | 설명 |
|-----------|------|
| 제목 | 매물명 (예: 옥정호수공원 인근 아파트 급매) |
| 주소 | 매물 주소 |
| 건물 종류 | 아파트 / 상가 / 오피스텔 등 |
| 거래 방식 | 매매 / 전세 / 월세 |
| 가격 | 매매가 또는 보증금 (만원 단위) |
| 월세 | 월세인 경우 월 임대료 (만원) |
| 공급 면적 | 공급 면적 (m²) |
| 방 수 / 욕실 수 | 숫자 입력 |
| 층수 / 전체층 | 숫자 입력 |
| 추천 매물 여부 | ✅ 체크 시 홈페이지 메인에 노출 |
| 설명 | 매물 상세 설명 |
| 이미지 | 최대 10장까지 업로드 가능 |

2. 모든 내용 입력 후 **[등록하기]** 버튼을 클릭합니다.

### 4-2. 매물 수정 및 삭제

- 매물 목록에서 각 매물 카드 우측의 **[삭제]** 버튼으로 삭제할 수 있습니다.
- 수정이 필요한 경우에는 해당 매물을 삭제 후 재등록하는 방식을 사용하세요.

### 4-3. 매물 보기 방식 (리스트/카드 전환)

홈페이지 매물 목록 화면에서 우측 상단의 **[리스트 / 카드]** 버튼으로 보기 방식을 전환할 수 있습니다.

## 5. 상담 신청 및 답변 관리 (알림 문자 기능)

관리자 로그인 후 **[상담신청 관리]** 탭을 클릭합니다.

- 고객이 홈페이지에서 상담을 신청하면 이 목록에 자동으로 기록됩니다.
- 동시에 설정된 이메일 주소로 **자동 알림 이메일**이 발송됩니다.
- 대기 중인 신청 건의 우측 **[답변]** 영역을 눌러 답변 등록 및 알림 문자를 발송할 수 있습니다.

### 5-1. 상담 답변 작성 및 알림 문자 전송

상담 신청 건의 상태를 **대기중 ➡️ 답변완료**로 전환하며, 동시에 고객에게 알림 문자 메시지를 전송합니다. 
화면의 입력 폼은 다음과 같이 이원화되어 있습니다:

1. **상담 답변 (상세 내용 - 홈페이지용)**:
   * 고객이 홈페이지에서 본인의 접수정보로 조회를 수행할 때 화면에 노출되는 **정식 상세 답변**입니다.
   * 분량에 관계없이 상세히 적으셔도 됩니다.
2. **알림 문자 내용 (전송용 - 간단히 요약)**:
   * 실제 고객의 휴대폰으로 전송되는 **요약 안내 문자**입니다.
   * **자동 연동**: 상담 답변을 적기 시작하면, 문자 입력란에 첫 50글자가 자동으로 연동되어 미리 작성됩니다.
   * **기본 문구 채우기**: 우측의 링크를 누르면 *"문의하신 내용의 답변이 등록되었습니다. 아래 링크에서 확인해 주세요."*라는 범용 문구가 자동 세팅됩니다.
   * **단축 답변 버튼**: 상단의 `[방문 유도]`, `[유선 요청]`, `[매물 소진]` 버튼을 누르면 홈페이지용 상세 답변과 문자 전송용 안내 문구가 용도에 맞게 각각 최적화된 서식으로 동시에 세팅됩니다.

### 5-2. 알림 문자(SMS/LMS) 전송 템플릿 규격

문자 전송 시 고객 휴대폰에는 아래의 형태로 깔끔하게 전달됩니다:

```text
[리더가든부동산의 답변입니다.]
(작성한 간단한 문자 요약)

상세 답변 확인:
https://내홈페이지주소.com/consult/view
```

* **단/장문 자동 감지**: 글자 수가 늘어나더라도 시스템이 알아서 단문(SMS)과 장문(LMS)을 판단하여 자동으로 변환 발송합니다.

> 🔒 **보안 안내 (환경 변수 비노출)**:
> 문자 발송을 위한 솔라피 API Key 및 Secret 등 민감 정보는 사용자 브라우저 소스 코드를 통해 노출되는 로컬 환경설정 파일(`.env` 또는 `.env.local`)에 절대 적지 않습니다.
> 이 정보들은 오직 안전한 서버 공간인 **Supabase Edge Function Secrets** 내부에서만 안전하게 암호화되어 관리됩니다.

## 6. 기준정보 관리 (공통코드)

관리자 로그인 후 **[기준정보 관리]** 탭을 클릭합니다.

매물 등록 시 사용하는 **건물 종류**, **거래 방식** 등의 선택 항목을 추가·삭제할 수 있습니다.

### 현재 등록된 기준정보

| 분류 | 예시 항목 |
|------|-----------|
| 건물 종류 | 아파트, 오피스텔, 상가, 빌라, 토지 등 |
| 거래 방식 | 매매, 전세, 월세 |
| 메인 슬로건 예시 | 슬로건 템플릿 목록 |
| 서브 슬로건 예시 | 서브 슬로건 템플릿 목록 |

### 항목 추가 방법

1. 분류 선택 (건물 종류 / 거래 방식 / 슬로건 등)
2. 코드값(영문) 입력 (예: `VILLA`)
3. 표시이름(한글) 입력 (예: `빌라/연립`)
4. 정렬순서 입력 (숫자가 작을수록 먼저 표시)
5. **[등록]** 버튼 클릭

### 항목 삭제

각 항목 우측의 🗑️ 아이콘을 클릭하면 삭제됩니다.  
> ⚠️ 이미 매물에 사용된 코드는 삭제해도 기존 매물에는 영향 없습니다.

## 7. 신규 중개소 초기 세팅 가이드

**이 홈페이지를 새로운 중개소에 납품할 때** 순서대로 진행하세요.

### Step 1. Supabase 데이터베이스 세팅

다음 SQL 스크립트를 Supabase **SQL 에디터**에서 순서대로 실행합니다:

```
1. setup_platform_db.sql       (종합 스키마 설정 및 complexes 테이블 등 생성)
2. setup_consultations.sql     (상담신청 테이블)
3. setup_storage.sql           (이미지 저장소)
4. setup_agencies_extended.sql (사이트 설정 컬럼)
5. setup_agencies_hours.sql    (영업시간 컬럼)
6. setup_slogan_codes.sql      (슬로건 예시 데이터)
```

### Step 2. agencies 테이블에 중개소 기본 정보 입력

Supabase ➡️ Table Editor ➡️ `agencies` 테이블에서 새 행을 추가합니다:

| 컬럼 | 값 |
|------|----|
| id | (자동 생성 UUID) |
| name | 중개소 상호명 |
| ceo_name | 대표자명 |
| phone | 고객 노출용 대표 연락처 |
| notification_phone | 문자 알림 수신용 핸드폰 번호 |
| email | 메일 알림 수신용 이메일 주소 |
| registration_number | 개설등록번호 |
| address | 사무실 주소 |

### Step 3. 환경변수(.env) 파일 설정

```env
VITE_SUPABASE_URL=https://[프로젝트ID].supabase.co
VITE_SUPABASE_ANON_KEY=[익명 키]
VITE_ADMIN_PASSWORD=[관리자 비밀번호]
```

### Step 4. 관리자 페이지에서 나머지 정보 입력

`/admin` 접속 ➡️ [사이트 환경 설정] 탭에서:
1. 기본 정보 (상호명, 대표자, 대표 연락처, 알림 수신 번호, 알림 이메일, 등록번호, 주소)
2. 이미지 업로드 (명함, 배경 사진)
3. 슬로건 문구 입력
4. 영업시간 설정
5. API 키 입력 (네이버 지도, EmailJS)
6. **[환경 설정 저장하기]** 클릭

### Step 5. 기준정보 등록

[기준정보 관리] 탭에서 해당 중개소에 맞는 건물 종류, 거래 방식 항목을 추가합니다.

### Step 6. 첫 매물 등록 및 최종 확인

[매물 관리] 탭에서 대표 매물 1~2개를 등록하고 홈페이지가 정상 표시되는지 확인합니다.

### Step 7. Supabase Secrets 및 Edge Function (알림 문자 설정)

고객 상담 답변 알림 문자가 전송되도록 문자 전송 모듈을 설정합니다.

1. **Edge Function 배포**:
   개발 컴퓨터 터미널에서 다음 명령어를 실행합니다.
   ```bash
   npx supabase login
   npx supabase functions deploy send-reply-sms --project-ref [프로젝트ID]
   ```
2. **비밀키(Secrets) 등록**:
   Supabase 대시보드 ➡️ **[Edge Functions]** ➡️ **[Secrets]** 메뉴로 이동하여 다음 3가지를 추가합니다:
   * **`SOLAPI_API_KEY`**: 솔라피 API Key
   * **`SOLAPI_API_SECRET`**: 솔라피 API Secret
   * **`SOLAPI_SENDER_NUMBER`**: 솔라피에 사전 승인 등록된 발신번호 (하이픈 없이 숫자만 입력)
   * ⚠️ **보안 경고**: 이 키값들은 고객 브라우저 소스코드 검사를 통해 외부로 노출될 위험이 크므로, 로컬 컴퓨터의 `.env` 파일에 절대 기록하지 마십시오. 반드시 Supabase Secrets 영역에만 등록해 관리하십시오.

## 8. 자주 묻는 질문 (FAQ)

**Q. 이미지를 올렸는데 홈페이지에 바로 안 보여요.**  
A. [환경 설정 저장하기] 버튼을 눌렀는지 확인하세요. 저장 후 홈페이지를 새로고침(F5)하면 반영됩니다.

**Q. 특별 공지사항을 지우고 싶어요.**  
A. 특별 공지사항 텍스트박스 내용을 모두 지우고 저장하면 자동으로 사라집니다.

**Q. 슬로건에 줄바꿈을 하고 싶어요.**  
A. 텍스트박스 안에서 Enter 키를 누르면 홈페이지에도 그대로 줄바꿈이 적용됩니다.

**Q. 매물 이미지가 10장을 넘어요.**  
A. 현재 최대 10장까지 지원합니다. 가장 잘 나온 사진 위주로 선별해서 올려주세요.

**Q. 상담 신청 이메일이 안 와요.**  
A. [사이트 환경 설정] → [API 환경 변수 세팅]에서 EmailJS 키 3개가 올바르게 입력되어 있는지 확인하세요.

**Q. 지도에 사무소 위치가 잘못 표시돼요.**  
A. 네이버 지도에서 사무소 위치를 검색한 후 정확한 위도/경도를 확인하여 DB에서 수정이 필요합니다. (시스템 관리자 문의)

## 10. 시스템 상태 진단 및 자가 해결 가이드

홈페이지가 예기치 않게 로딩되지 않거나, 지도/이메일 연동에 에러가 생기는 경우 관리자가 직접 진단하고 임시/영구 해결할 수 있는 **자가 진단 화면**을 지원합니다.

### 10-1. 진단 화면 접속 방법
* 홈페이지 하단 푸터(Footer) 영역의 오른쪽 밑에 있는 **`[🛠️ 시스템 상태 진단 및 자가 해결]`** 바로가기 링크 버튼을 클릭하거나 브라우저 주소창에 `http://localhost:5173/diagnostics`를 입력하여 접속합니다.

### 10-2. 주요 진단 기능 및 해결 방안

#### 1) 인터넷 연결 상태
* **점검**: `navigator.onLine` 상태를 확인합니다.
* **해결**: 인터넷이 끊겨 오프라인으로 뜰 경우 회선/공유기를 점검하고, 정상 복구된 후 카드 우측의 **`[🔄 새로고침]`** 버튼을 눌러 상태를 갱신합니다.

#### 2) 환경 변수 검사 및 임시 오버라이드
* **점검**: `.env`에 정의된 API Key 및 Supabase 설정(6개 항목) 로드 여부를 검증합니다.
* **자가 해결**: 
  * **[임시 환경 변수 설정]** 탭에서 임시로 정상 작동하는 다른 API 키나 Supabase 주소를 브라우저상에서 임시 대입하여 실시간 테스트할 수 있습니다.
  * 테스트 완료 후 **`[📋 .env 파일 내용 복사]`** 버튼을 눌러 클립보드에 복사한 후, 프로젝트 내 `.env` 파일에 덮어씌워 저장하면 영구 적용됩니다.

#### 3) Supabase DB 테이블 검사
* **점검**: `agencies`, `properties_v2`, `common_codes`, `consultations` 등 4가지 필수 테이블의 연결/조회 및 데이터 양을 점검합니다.
* **자가 해결**:
  * 테이블 조회 실패(테이블 미생성) 시, 해당 항목 밑에 노출되는 **`[🛠️ SQL 생성 스크립트 복사하러 가기]`** 버튼을 클릭하여 SQL을 복사합니다.
  * Supabase 대시보드의 **SQL Editor**에 복사한 코드를 붙여넣고 **[Run]**을 실행하여 즉시 테이블을 생성할 수 있습니다.

#### 4) 네이버 지도 인증 점검 (VPC 신규 API 대응)
* **점검**: 네이버 지도 스크립트 객체(`window.naver`) 마운트 여부와 실제 지도 컴포넌트 작동을 시각적으로 점검합니다.
* **자가 해결 (Client ID / URL 오류)**:
  * 네이버 클라우드 플랫폼의 애플리케이션 서비스 환경 설정 시 Web 서비스 URL 뒤에 슬래시(`/`) 유무에 따라 매칭 에러가 발생하므로, 슬래시가 포함된 주소와 제외된 주소를 모두 등록해 주어야 합니다.
    * 등록 주소 예시: `http://localhost:5173`, `http://localhost:5173/` 둘 다 등록
  * **신규 발급 API 인증**: 최근 가입한 애플리케이션(VPC 환경)의 경우 호출 시 `ncpClientId` 대신 최신 규격인 **`ncpKeyId`** 파라미터를 사용해 호출해야 하므로, 코드 내부적으로 `ncpKeyId`를 사용하도록 패치되었습니다.

#### 5) EmailJS 실시간 이메일 전송 테스트
* **점검**: EmailJS 연동 Key의 정상 매칭 여부를 실시간 테스트 발송을 통해 점검합니다.
* **자가 해결**: 이메일 입력 폼에 메일 주소를 입력한 후 **`[📨 테스트 메일 전송]`** 버튼을 클릭하여 API가 수신하는 실시간 JSON 통신 로그(성공 `200` 또는 실패 에러 코드)를 직접 보며 디버깅할 수 있습니다.
